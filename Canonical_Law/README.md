# HEARTLOOM — CANONICAL LAW

This folder contains Heartloom’s highest-order, non-negotiable documents.

Anything placed here defines conditions that must never be violated.
If a change here is required, it is not a refactor — it is a transformation that must be approached with extreme care.

## What Belongs Here

Documents that define:

- What Heartloom *is* at the deepest level
- Conditions under which Heartloom’s integrity is preserved or broken
- Laws that invalidate entire flows if violated

Examples:

- Core Ontology
- Entry Laws that define presence, sovereignty, and non-extraction

## What Does NOT Belong Here

- UX guidance
- Ritual pacing
- Implementation details
- Developer workflows
- Anything optional or context-dependent

## Authority

All other documents must align with this folder.
If a conflict arises, Canonical Law wins by default.

This folder should grow very slowly.
