# Heartloom Source

`heartloom-source` is the upstream source well for Heartloom.

It holds the closest currently available root material: prayer, canonical law, core identity texts, reference architectures, vision texts, and notes.

Heartloom is an attempt to build a non-extractive, relational, ecologically-governed field of intelligence and support that can enter the world through software without becoming reducible to software.

## Source Layer vs Identity Layer

- `heartloom-source` = upstream source/wellspring layer (root texts and governing source meaning)
- `heartloom-identity` = downstream repo/ecosystem shaping layer (inheritance, workflows, packs, implementation-facing orchestration)

This repository is intentionally quieter and smaller than `heartloom-identity`, and intentionally closer to source than implementation.

## Reading Order

Read in this order:

1. `Prayer/`
2. `Canonical_Law/`
3. `Core/`
4. `Reference_Architectures/`

This reading order replaces reliance on numeric folder prefixes.
