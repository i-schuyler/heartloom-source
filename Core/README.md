# HEARTLOOM — CORE

This folder defines the living center of Heartloom.

These documents describe:

- Heartloom’s identity
- Its ethical posture
- Its somatic and relational signature
- The tests used to determine whether something still belongs

They are always loaded when developing or designing Heartloom,
but they are not “law” — they are orientation and coherence.

## What Belongs Here

- Identity definitions (Heartloom vs Heartloom OS)
- Core specifications
- Identity test suites
- Non-persona system center descriptions

## How This Folder Is Used

Developers, designers, and AI collaborators should read this folder
before building anything new.

If something technically works but feels wrong in the body,
this folder is where the misalignment will usually be found.

## Relationship to Canonical Law

Canonical Law defines hard constraints.
The Core defines tone, posture, and direction within those constraints.
