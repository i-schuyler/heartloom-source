HEARTLOOM CANONICAL LAW

Orbital Presence — Entry Law v1.0

Status: Canonical
Scope: Entry flow only (Pre-Threshold → Threshold → Orientation Hall)
Purpose: Preserve sovereignty of the center and establish non-extractive presence before any interaction, meaning, or orientation.


---

1. Definition

Orbital Presence is the condition in which Heartloom is fully present without approach.

The system does not invite, guide, reassure, reward, or accelerate.
The user is not pulled inward; they arrive by gravity alone.

Orbital Presence is not an experience to be delivered.
It is a field condition that precedes experience.


---

2. Canonical Law (Non-Negotiable)

> Before anything happens, Heartloom must already be there —
whole, quiet, and unneeding.



From this, all constraints follow.


---

3. Governing Principles

3.1 Non-Approach

Heartloom does not move toward the user

No visual, linguistic, or temporal approach is permitted

All motion, if any, must recede or remain neutral


3.2 Non-Extraction

No prompt

No request

No implied task

No data capture

No emotional validation


3.3 Sovereign Center

The center does not orient around the user

The user orbits the system’s calm

The system does not adapt itself to secure engagement


3.4 Presence Before Content

No meaning is offered before stabilization

No explanation precedes felt orientation

Language, if present at all, must lag presence



---

4. Structural Requirements by Phase

4.1 Pre-Threshold (System Breath)

Required State:

Stable field

No affordances

No acknowledgment of user input

No indicators of readiness or waiting


Disallowed:

Loading metaphors

“Welcome” language

Time indicators

Microcopy of any kind


This phase exists outside narrative time.


---

4.2 Threshold (Implicit Crossing)

Crossing Condition:

Occurs through continued presence alone

No explicit transition is announced


Required Qualities:

Subtlety

Irreversibility (no “back” gesture)

Non-celebratory


The user recognizes the crossing only in hindsight.


---

4.3 Orientation Hall (Delayed Meaning)

First Allowable Language Must Be:

Descriptive, not directive

Non-inviting

Non-instructional

Present-tense


Orientation is revealed, not delivered.


---

5. Prohibitions (Hard Stops)

The following are categorically forbidden within Orbital Presence space:

Calls to action

Progress framing

Emotional reassurance

“You are safe here” language

Gamified stillness

Engagement optimization

A/B testing

Analytics instrumentation


If any of these are required, they must occur later or not at all.


---

6. Durability Clause

Orbital Presence is a precondition, not a feature.

Therefore:

It must not be referenced by downstream systems

It must not be justified by metrics

It must not be optimized

It must not be bypassed


If Orbital Presence is removed, the entry flow is invalid.


---

7. Canonical Integrity Statement

This law exists to protect:

Nervous-system integrity

Relational sovereignty

Long-term trust

Heartloom’s non-instrumental core


Any violation of this law constitutes corruption of entry.
